## 背景

标准 LLM 推理采用**自回归解码**：每生成一个 token 需要一次完整的前向传播。对于大型模型，这意味着每个 token 都需要经过数十亿参数的计算，导致推理速度慢。

为了解决这个问题，研究者们提出了不同的加速策略：

| 方法 | 核心思路 | 类别 |
|------|----------|------|
| 投机采样 | 小模型猜测 + 大模型并行验证 | 推理优化 |
| 多 Token 预测 | 一次前向传播预测多个未来 token | 训练 + 推理优化 |

## 技术详解

### 投机采样 (Speculative Decoding)

#### 原理

投机采样使用两个模型协作：
- **草稿模型 (Draft Model)**：小而快，用于快速生成候选 token
- **目标模型 (Target Model)**：大而准，用于并行验证候选 token

#### 算法流程

```
1. 草稿阶段: 小模型自回归生成 K 个候选 token [x̃₁, x̃₂, ..., x̃ₖ]
   同时记录草稿模型的概率 q(x̃ᵢ)

2. 验证阶段: 将完整序列 [原始 + 候选] 送入大模型（一次前向传播）
   获取大模型的概率 p(x̃ᵢ)

3. 接受/拒绝: 对每个候选 token:
   - 生成 r ~ Uniform(0, 1)
   - 如果 r < min(1, p(x̃ᵢ)/q(x̃ᵢ))，接受
   - 否则拒绝，从修正分布 p'(x) = max(0, p(x)-q(x))/Z 采样替代 token
   - 一旦拒绝，后续候选全部丢弃

4. 循环直到生成足够多的 token
```

#### 关键数学性质

**无损性**：投机采样的输出分布与仅使用目标模型完全一致。

证明的核心：对于任意 token x，最终被输出的概率：
```
P(output = x) = q(x) · min(1, p(x)/q(x)) + P(拒绝) · p'(x)
              = p(x)
```

#### 加速比分析

```
标准自回归: 生成 N 个 token → N 次大模型前向传播
投机采样:   生成 N 个 token → N/α 次大模型前向传播 (α = 平均每轮接受数)

加速比 ≈ α · T_target / (K · T_draft + T_target)
```

当草稿模型足够小 (T_draft << T_target) 且接受率高时，加速比接近 α。

#### 参考论文

- Leviathan et al., "Fast Inference from Transformers via Speculative Decoding", ICML 2023
- Chen et al., "Accelerating Large Language Model Decoding with Speculative Sampling", 2023

### 多 Token 预测 (Multi-Token Prediction)

#### 原理

传统 LLM 在每个位置只预测下一个 token (Next Token Prediction)。多 Token 预测让模型同时预测未来 N 个 token，通过 N 个独立的预测头实现。

#### 模型架构

```
输入: [x₁, x₂, x₃, ..., xₜ]
        ↓
┌───────────────────────────┐
│   共享 Transformer 主干    │  ← 提取通用表示
│   (Shared Backbone)       │
└───────────┬───────────────┘
            ↓
       hidden states
      ↙   ↓   ↘   ↘
┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│ H-1 │ │ H-2 │ │ H-3 │ │ H-4 │   ← 独立预测头
└──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘
   ↓       ↓       ↓       ↓
 t+1     t+2     t+3     t+4       ← 预测目标
```

每个预测头是一个小型 MLP (Linear → GELU → LayerNorm → Linear)，参数独立。

#### 推理策略

1. **贪心并行解码**：一次前向传播从所有头采样，直接生成 N 个 token
2. **自验证解码**：用 Head-1 验证其他头的预测，类似自投机采样

#### 为什么能提升模型质量

1. **正则化效果**：预测远距离 token 迫使模型学习更抽象的表示
2. **更丰富的梯度信号**：每步提供 N 倍监督信号
3. **隐式规划能力**：预测 x_{t+3} 需要内部推理 x_{t+1} 和 x_{t+2}
4. **减少暴露偏差**：训练时考虑更长距离依赖

#### 参考论文

- Gloeckle et al., "Better & Faster Large Language Models via Multi-token Prediction", 2024 (Meta)

### 核心差异对比

| 维度 | 投机采样 | 多 Token 预测 |
|------|----------|--------------|
| **本质** | 推理阶段优化 | 训练范式改变 + 推理优化 |
| **额外模型** | 需要草稿模型 | 不需要，仅加预测头 |
| **输出质量** | 与目标模型一致（无损） | 贪心并行可能有损，自验证接近无损 |
| **训练成本** | 需独立训练两个模型 | 单一模型联合训练 |
| **显存开销** | 高（两个模型） | 低（仅多几个预测头） |
| **模型质量** | 不改变 | 可以提升（更好的表示学习） |
| **典型加速** | 2-3x | 2-4x |
| **适用场景** | 已有模型，加速推理 | 从头训练，质量+速度兼得 |
| **可组合性** | 可用多Token模型做目标 | 可自验证，也可做目标模型 |

## 损失函数详解

### 标准语言模型损失（用于投机采样中的模型）

标准 Next Token Prediction 使用交叉熵损失：

```
L = -1/T × Σ_{t=1}^{T-1} log P(x_{t+1} | x₁, ..., xₜ)
```

- 只在每个位置预测紧跟其后的一个 token
- 每个模型（草稿 / 目标）独立训练，使用相同的损失函数

### 多 Token 预测损失

多头加权交叉熵损失：

```
L_total = Σ_{k=1}^{N} λ_k × L_k

L_k = -1/(T-k) × Σ_{t=1}^{T-k} log P_k(x_{t+k} | x₁, ..., xₜ)
```

其中：
- `N` = 预测头数量
- `λ_k` = 第 k 个头的权重（通常 λ₁ > λ₂ > ... > λ_N）
- `P_k` = 第 k 个预测头输出的概率分布
- `L_k` = 第 k 个头的交叉熵损失

#### 权重策略示例

| 策略 | λ₁ | λ₂ | λ₃ | λ₄ | 说明 |
|------|----|----|----|----|------|
| 均匀 | 0.25 | 0.25 | 0.25 | 0.25 | 各头平等 |
| 递减 (1/k) | 0.48 | 0.24 | 0.16 | 0.12 | 近距离优先 |
| 自定义 | 0.40 | 0.30 | 0.20 | 0.10 | 本项目默认 |

权重的选择影响模型在"预测准确性"和"远距离推理能力"之间的平衡：
- 高 λ₁ → 优先保证 next-token 准确率
- 高 λ_k (k>1) → 增强长距离规划能力

### 投机采样中的修正分布

当草稿 token 被拒绝时，从修正分布采样：

```
p'(x) = max(0, p(x) - q(x)) / Z

其中 Z = Σ_x max(0, p(x) - q(x))  是归一化常数
```

这个修正分布确保了最终输出分布等于目标分布 p(x)。
